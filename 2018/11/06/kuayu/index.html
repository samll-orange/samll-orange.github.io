跨域

发表于 2018-11-06 | 更新于 2018-11-12 | 分类于 JavaScript

首先先分析url构成
一个域名地址的组成：

如：
http://www.abc.com:8080/scripts/jQuery.js

协议：http
子域名：www
主域名：abc.com
端口：8080
请求资源：scripts/jQuery.js

什么是跨域？

定义：
由于“同源策略”的限制造成的：同一协议、同一域名、同一端口
当协议、子域名、主域名、端口号中的任意一个不相同时，都算作不同域。
不同域之间互相请求资源，就算作“跨域”。

比如：http://www.abc.com/index.html 请求 http://www.efg.com/service.php

总结跨域的情况如下：

1. 子域名相同、主域名相同、端口号相同 -- 非跨域


2. 子域名相同、主域名不同、端口号相同 -- 跨域


3. 子域名不同、主域名相同、端口号相同 -- 跨域


4. 子域名相同、主域名相同、端口号不同 -- 跨域


 5.协议不同（http、https、file）、子域名相同、主域名相同、端口号相同 -- 跨域

  • 通过在同域名的web服务器端创建一个代理：
  • 广州服务器：域名 www.guangzhou.com
  • 深圳服务器：域名 www.shenzhen.com
  • 如：
    深圳服务器后台：
